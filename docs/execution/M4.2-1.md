## Work Log

### 1. `scripts/validate.sh`

Created [scripts/validate.sh](../../scripts/validate.sh) as the canonical
"validate everything" entry point per
[coding-guidelines.md](../policies/common/coding-guidelines.md). The script
runs all declared quality gates in sequence:

1. `ruff check src/ tests/`
2. `ruff format --check src/ tests/`
3. `pyright`
4. `pytest`

Uses `set -euo pipefail` so execution stops on the first failure. Invokes
tools through `.venv/bin/` to match the repository's documented command
convention.

### 2. Pyright configuration

- Added `pyright>=1.1` to `[project.optional-dependencies].dev` in
  [pyproject.toml](../../pyproject.toml).
- Added `[tool.pyright]` section with `pythonVersion = "3.13"`,
  `typeCheckingMode = "standard"`, `include = ["src"]`.
- `standard` mode produces zero errors on the clean tree. Tightening to
  `strict` was not attempted since the ticket notes specify `standard` as the
  starting baseline.

### 3. PEP 561 `py.typed` marker

### 4. Semantic types (`_types.py`)

Created [src/context_sync/_types.py](../../src/context_sync/_types.py) with
`NewType` aliases for all identified domain concepts:

| Alias | Runtime type | Domain meaning |
| --- | --- | --- |
| `IssueId` | `str` | Stable Linear issue UUID |
| `IssueKey` | `str` | Human-facing issue key (e.g. `"ACP-123"`) |
| `WorkspaceId` | `str` | Immutable Linear workspace UUID |
| `WorkspaceSlug` | `str` | Human-readable workspace slug |
| `CommentId` | `str` | Stable Linear comment UUID |
| `AttachmentId` | `str` | Stable Linear attachment UUID |
| `WriterId` | `str` | Unique writer-lock invocation identifier |

All seven types are re-exported from
[`__init__.py`](../../src/context_sync/__init__.py) and listed in `__all__`.
