### Key design decisions

1. **Reuse `_refresh_under_lock` rather than inline refresh logic.** The `add`
   and `remove_root` methods modify the manifest, persist it to disk, then
   delegate the whole-snapshot refresh to `_refresh_under_lock` which re-loads
   the manifest. This avoids duplicating the refresh pipeline.

2. **Always fetch during `add` to validate availability.** Even when alias
   resolution succeeds, `add` fetches the ticket to confirm it is still
   available and to validate workspace membership. The resolved UUID is used as
   the fetch key when available (avoids redundant key‚ÜíUUID resolution).

3. **Zero-roots pruning.** When `_refresh_under_lock` encounters an empty root
   set, it now prunes all remaining tracked tickets and their files rather than
   returning a vacuous empty result. This is correct: if no roots exist,
   nothing is reachable, so all tracked tickets should be removed.

## Phase B ‚Äî Review Response

All six findings from the two review passes were accepted and addressed.

### M3-2-R1 ‚Äî Alias precedence inversion (Medium)

**Response:** Accept. Inverted the resolution order in `_resolve_ref_to_uuid`
so `current_key` matches take precedence over historical aliases.  This aligns
with the Tier 3 URL resolver in `_pipeline.py` and the ADR ¬ß1.6 precedence
rationale.

**Changes:**
- [src/context_sync/_ticket_ref.py](../../src/context_sync/_ticket_ref.py):
  Resolution order is now (1) `current_key`, (2) aliases, (3) direct UUID.
- [tests/test_add_remove_root.py](../../tests/test_add_remove_root.py):
  `TestAliasCurrentKeyPrecedence` ‚Äî regression test injecting a conflicting
  historical alias and asserting the current-key owner wins.

### M3-2-R2 ‚Äî Pre-refresh partial commit (High)

**Response:** Accept. `_refresh_under_lock` now accepts an optional
pre-loaded `manifest` parameter. `add` and `remove_root` pass the in-memory
manifest rather than saving-then-reloading, so the root-set mutation and
snapshot finalization are committed together in the final `save_manifest` call.

**Changes:**
- [src/context_sync/_sync.py](../../src/context_sync/_sync.py):
  Added `manifest` parameter to `_refresh_under_lock`. Removed the
  pre-refresh `save_manifest` calls from `_add_under_lock` and
  `_remove_root_under_lock`.
- [tests/test_add_remove_root.py](../../tests/test_add_remove_root.py):
  `TestNoPartialCommitOnRefreshFailure` ‚Äî verifies atomic commit semantics.

### M3-2-R3 ‚Äî Derived-ticket UUID resolution gap (Medium)

**Response:** Accept. Step 3 of `_resolve_ref_to_uuid` now checks
`manifest.tickets` (all tracked tickets) rather than only `manifest.roots`,
so derived-ticket UUIDs resolve correctly and reach the specific "not in the
root set" error.

**Changes:**
- [src/context_sync/_ticket_ref.py](../../src/context_sync/_ticket_ref.py):
  Step 3 checks `manifest.tickets` then falls back to `manifest.roots`.
- [tests/test_add_remove_root.py](../../tests/test_add_remove_root.py):
  `TestRemoveRootByDerivedUuid` ‚Äî tests both derived-UUID and root-UUID
  inputs to `remove_root`.

### M3-2-R4 ‚Äî Missing INFO log on zero-roots prune (Low)

**Response:** Accept. Added an INFO-level summary log before the early return
in the zero-roots branch, reporting the snapshot mode, prune count, and
duration.

**Changes:**

  `logger.info` call in the zero-roots early-return path.

### M3-2-R5 ‚Äî File size extraction (Low)

**Response:** Accept. Extracted `_parse_linear_url`, `_normalize_ticket_ref`,
`_resolve_ref_to_uuid`, and `_LINEAR_URL_RE` into a new
