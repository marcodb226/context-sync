- Designed the public API:
  - `TicketWriteResult` — frozen dataclass recording the outcome of one
    `write_ticket` call: `issue_id`, `issue_key`, `previous_key` (the key
    before an issue-key rename, or `None`), `file_path` (relative to
    context dir).
  - `compute_refresh_cursor(bundle)` — derives `RefreshCommentMeta` from
    `CommentData` using `resolve_root_comment`, derives `RefreshThreadMeta`
    from `ThreadData`, and returns the v1 composite cursor dict with keys
    `issue_updated_at`, `comments_signature`, `relations_signature`.
  - `fetch_tickets(issue_ids, *, gateway, semaphore)` — uses
    `asyncio.TaskGroup` with the shared semaphore for bounded concurrent
    fetches; propagates exceptions from the task group to the caller.
  - `write_ticket(bundle, *, root_state, quarantined_reason=None,
    last_synced_at, context_dir, manifest)` — synchronous; detects
    issue-key renames by comparing `bundle.issue.issue_key` to
    `manifest.tickets[issue_id].current_key`; records old key in
    `manifest.aliases`, renames the old file if it exists, updates
    `manifest.tickets`, computes the refresh cursor, renders and writes via
    `render_ticket_file` + `write_and_verify_ticket`.
  - `make_ticket_ref_provider(fetched, *, gateway, semaphore)` — returns a
    `TicketRefProvider` that scans description and comment bodies of
    already-fetched tickets for `https://linear.app/*/issue/{KEY}` URLs,
    resolves unknown keys via the gateway (mutating `fetched` as a side
    effect), and deduplicates results before returning them.
- Chose `_LINEAR_ISSUE_KEY_RE = re.compile(r"https://linear\\.app/[^/\\s]+/issue/([A-Z][A-Z0-9]*-\\d+)")`

## Coding Guidelines Compliance

Checked against
[docs/policies/common/coding-guidelines.md](../policies/common/coding-guidelines.md)
and
[docs/policies/common/coding-guidelines-python.md](../policies/common/coding-guidelines-python.md):

- **Documentation**: Module has a header docstring. All public types and
  functions have docstrings with Parameters/Returns/Raises sections.
  Private helpers are documented.
- **No magic numbers**: No bare literals. `_LINEAR_ISSUE_KEY_RE` is a named
  module-level constant.
- **Type annotations**: All parameters and return types annotated. Frozen
  dataclass for `TicketWriteResult`. `TicketRefProvider` alias reused from
  `_traversal`.
- **Data modeling**: `TicketWriteResult` is a frozen dataclass (internal
  state container from trusted code). `Manifest` is Pydantic (serialization
  boundary); its dict fields are mutated in place by `write_ticket`.
- **Async**: `fetch_tickets` and the provider returned by
  `make_ticket_ref_provider` are async. `write_ticket` is synchronous;
  it only calls synchronous I/O helpers.
- **DRY**: `compute_refresh_cursor` reuses `resolve_root_comment` from
  `_renderer` and the signature functions from `_signatures`. The URL scanner
  is extracted into `_extract_issue_keys` and `_bundle_content_texts`.
- **Fail loudly**: Gateway and signature exceptions propagate unchanged.
  `write_and_verify_ticket` raises `WriteError` on mismatch; this is
  documented in `write_ticket`'s docstring.
- **Logging**: `DEBUG`-level logs for issue-key rename detection, file
  rename, and unresolvable ticket-ref keys.
- **No hardcoded secrets**: None — this module reads and writes graph
  topology and ticket content only.
- **Stable-release changelog gate**: No entry required — pre-`1.0.0`
  repository (`0.1.0.dev0`).
