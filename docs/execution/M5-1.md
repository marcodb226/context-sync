### Follow-up items

- The `_types.py` module still defines independent `NewType` aliases for `IssueId`, `IssueKey`, `CommentId`, `AttachmentId`, and `Timestamp` rather than re-exporting from `linear_client.types`. The M5-D1 audit recommends upstream re-export for shared NewType identity, but `linear-client` is not a declared `pyproject.toml` dependency (it's a workspace sibling). The gateway uses `UpstreamIssueId` imports at the boundary to satisfy Pyright. Full type unification would require adding `linear-client` as a formal dependency or an optional dependency with TYPE_CHECKING-based re-exports.
- Coverage of the real `LinearNotFoundError` → `RootNotFoundError` mapping path (lines 297-298, 316-318, 408-411) requires the actual `linear_client.errors` module at test time. These are exercised implicitly by integration tests that use the real library but are not part of the unit test surface.
