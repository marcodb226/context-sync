- Chose module name `_traversal.py` for the new traversal engine, following
  the existing `_`-prefixed private-module convention.
- Designed the traversal result types as frozen dataclasses per the
  data-modeling policy:
  - `TraversedTicket` — per-ticket result holding UUID, key at discovery,
    effective depth (min across all roots), and the set of root UUIDs that
    can reach it.
  - `TraversalResult` — the full graph result: per-root reachable sets,
    the global union map, and the set of roots that hit the per-root cap.
- Designed `TicketRefProvider` as a `type` alias for the async callable
  injected by the sync flow (M2-3) for Tier 3 (`ticket_ref`) edge discovery.
  This keeps M2-1 free of any content-fetch dependency while still allowing
  M2-3 to inject URL scanning. When `None`, `ticket_ref` edges are silently
  skipped.
- Designed the core traversal algorithm `_traverse_single_root`:
  - BFS from one root; depth measured from that root.
  - Per-depth level: Tier 1+2 relations batch-fetched once from
    `gateway.get_ticket_relations`; Tier 3 (if provider given) called once
    per depth.
  - Tiers processed in priority order (Tier 1 → Tier 2 → Tier 3) at each
    depth so structural edges win over informational edges when the cap is
    near.
  - `at_cap=True` only when a ticket cannot be added due to the cap — not
    when the cap is merely reached after a successful add.
  - Cycle safety via a per-root `visited_depth` dict: once a UUID is added,
    it is never added again.
- Designed `build_reachable_graph` as the public entry point:
  - Iterates roots sequentially (ADR permits interleaving for efficiency;
    sequential is simpler and equally correct for the test surface).
  - Merges per-root results into a global union, tracking minimum depth and
    all root UUIDs per ticket.
  - Returns a `TraversalResult` with per-root sets, the union map, and the
    `roots_at_cap` frozenset.

## Coding Guidelines Compliance

Checked against
[docs/policies/common/coding-guidelines.md](../policies/common/coding-guidelines.md)
and
[docs/policies/common/python/coding-guidelines.md](../policies/common/python/coding-guidelines.md):

- **Documentation**: Module has a header docstring explaining design points.
  All public types and functions have docstrings with Attributes/Parameters/
  Returns/Raises sections. Private helpers are also documented.
- **No magic numbers**: No bare literals. `TRAVERSAL_TIERS`,
  `Dimension.TICKET_REF`, and the `_active_dims_for_tier` helper express
  all traversal constraints through named constants.
- **Type annotations**: All parameters and return types annotated. Frozen
  dataclasses for `TraversedTicket` and `TraversalResult`. `type` alias for
  `TicketRefProvider` (Python 3.12+ `type` statement per UP040).
- **Data modeling**: Frozen dataclasses for both result types per the
  data-modeling policy (internal state containers from trusted code).
- **Async**: `build_reachable_graph` and `_traverse_single_root` are
  `async`. Gateway and `ticket_ref_fn` calls use `await`. No blocking I/O.
- **DRY**: `_active_dims_for_tier` extracted as a shared helper used in
  both the `tier12_dims` computation and per-tier filtering. Relation-map
  fetch happens once per depth level and is reused for Tier 1 and Tier 2.
- **Fail loudly**: Exceptions from the gateway or `ticket_ref_fn` propagate
  unchanged to the caller (documented in the `build_reachable_graph`
  docstring).
- **Logging**: `DEBUG`-level log when a root hits its per-root cap, with
  the root UUID, cap value, tier context, and depth — enough to diagnose
  cap behaviour without a debugger.
- **No hardcoded secrets**: None — this module reads graph topology only.
- **Stable-release changelog gate**: No entry required — pre-`1.0.0`
  repository (`0.1.0.dev0`).

## Reviewer Handoff

- Confirm that `build_reachable_graph` produces independent per-root
  reachable sets with the root included in each set.
- Confirm that tier ordering (Tier 1 → Tier 2 → Tier 3) is enforced within
  each depth level and that the per-root cap stops expansion for
  lower-priority tiers when hit.
- Confirm that `effective_depth` in `TraversalResult.tickets` is the minimum
  depth across all roots, not the depth from the last root processed.
- Confirm that cycle safety holds: a ticket visited once is not revisited,
  and the traversal terminates even for cyclic relation graphs.
- Confirm that `roots_at_cap` contains only roots that actually failed to
  add at least one ticket, not roots that merely reached `cap_remaining==0`
  without a pending candidate.
- Confirm that `ticket_ref_fn=None` completely skips Tier 3 expansion without
  error and that `get_ticket_relations` is not called when all Tier 1+2
  dimensions are inactive at the current depth.
- Confirm that the module follows the adapter boundary from
  [docs/design/linear-domain-coverage-audit.md](../design/linear-domain-coverage-audit.md):
  Tier 1+2 edges come exclusively from `gateway.get_ticket_relations`; no
  direct `linear.gql.*` calls appear in this module.
