## Work Log

### 1. Coverage tooling

- Added `coverage[toml]>=7,<8` and `pytest-cov>=5,<6` to
  `[project.optional-dependencies].dev` in
  [pyproject.toml](../../pyproject.toml).
- Added `[tool.coverage.run]` and `[tool.coverage.report]` sections per
  [python/agent-awareness.md](../policies/common/python/agent-awareness.md):
  `branch = true`, `source = ["src"]`, `omit = ["src/*/version.py"]`,
  `show_missing = true`, `skip_covered = false`.
- Updated [scripts/validate.sh](../../scripts/validate.sh) to invoke
  `pytest --cov --cov-report=term-missing` instead of bare `pytest`,
  fulfilling the lockstep obligation from M4.2-1.

### 2. Ruff pydocstyle convention

- Added `[tool.ruff.lint.pydocstyle]` with `convention = "numpy"` to
  [pyproject.toml](../../pyproject.toml). Without this, ruff's auto-detection
  fell back to Google-style for methods without a `Parameters` section
  (specifically `diff()`), causing D416 false positives on standard numpy
  section headers.

### 3. Examples directory

  runnable script that exercises all four `ContextSync` public methods
  (`sync`, `refresh`, `diff`, `remove`) against a `FakeLinearGateway` in a
  temporary directory.
- The script uses `_gateway_override` with a comment explaining that the
  real constructor will be available once M5-1 lands the real gateway.
- The example builds a two-ticket graph (DEMO-1 root with a DEMO-2 child
  linked via relation edges) and demonstrates the complete sync/refresh/diff/
  remove lifecycle.
