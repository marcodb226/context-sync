- Implemented the full sync flow in
  [src/context_sync/_sync.py](../../src/context_sync/_sync.py):
  1. Resolve per-call dimension and cap overrides.
  2. Acquire writer lock with `acquire_lock(context_dir, "sync")`.
  3. Fetch the requested root ticket via `gateway.fetch_issue`.
  4. Load the existing manifest or initialize a new one from the root's
     workspace identity.
  5. Validate workspace: if the manifest exists and the root's workspace
     differs, raise `WorkspaceMismatchError` before any mutation.
  6. Update manifest traversal config (dimensions, max_tickets_per_root).
  7. Add the requested root to `manifest.roots` as `state="active"`.
  8. Pre-fetch all other active roots so the Tier 3 `ticket_ref` provider
     has their content for scanning at depth 0.
  9. Build the traversal roots dict `{uuid: current_key}` from the
     manifest root set and the fetched bundles.
  10. Build the `ticket_ref` provider via `make_ticket_ref_provider`.
  11. Build the reachable graph via `build_reachable_graph`.
  12. Fetch remaining reachable tickets not yet in the `fetched` dict.
  13. Write all reachable tickets via `write_ticket`, classifying each as
      `created` (new to manifest) or `updated` (already tracked).
  14. Prune derived tickets no longer reachable: delete local files,
      remove from `manifest.tickets`.
  15. Finalize snapshot metadata (`mode="sync"`, `completed_successfully=True`).
  16. Save the manifest via `save_manifest`.
  17. Release the writer lock in the `finally` block.

## Coding Guidelines Compliance

Checked against
[docs/policies/common/coding-guidelines.md](../policies/common/coding-guidelines.md)
and
[docs/policies/common/python/coding-guidelines.md](../policies/common/python/coding-guidelines.md):

- **Documentation**: Module docstring updated to reflect the sync
  implementation. `sync` method has a full docstring with Parameters, Returns,
  and Raises sections. `_sync_under_lock` has a docstring explaining its role.
  `_utc_now` has a one-line docstring.
- **No magic numbers**: No bare literals. `"sync"` string is used for the lock
  mode and snapshot mode, matching the `LOCK_MODES` literal type and the
  `ManifestSnapshot.mode` literal type.
- **Type annotations**: All parameters and return types annotated, including
  the `_sync_under_lock` private helper.
- **Data modeling**: No new data types introduced. Reuses `SyncResult`,
  `SyncError`, `ManifestRootEntry`, `ManifestSnapshot` from existing modules.
- **Async**: `sync` and `_sync_under_lock` are async. All gateway and pipeline
  calls use `await`. No blocking I/O in the async path. `write_ticket` is
  synchronous (documented in M2-2) and only calls synchronous I/O helpers.
- **DRY**: Reuses `build_reachable_graph`, `fetch_tickets`, `write_ticket`,
  `make_ticket_ref_provider`, `acquire_lock`, `release_lock`,
  `load_manifest`, `save_manifest`, and `initialize_manifest` from existing
  modules. No logic duplication.
- **Fail loudly**: Gateway exceptions propagate unchanged. `WorkspaceMismatchError`
  raised before any mutation. `WriteError` from `write_ticket` propagates.
  Lock contention errors from `acquire_lock` propagate.
- **Logging**: `INFO`-level logs at sync start (root count, cap) and
  completion (reachable count, created/updated/removed/error counts,
  roots at cap, duration). `DEBUG`-level log for derived-ticket pruning.
- **No hardcoded secrets**: None — this module orchestrates graph traversal
  and file I/O only.
- **Stable-release changelog gate**: No entry required — pre-`1.0.0`
  repository (`0.1.0.dev0`).
