#### Changes made

**R1 — Sync path consolidation**
([src/context_sync/_sync.py](../../src/context_sync/_sync.py)):

- `_sync_under_lock` now performs alias-based local resolution via
  `_normalize_ticket_ref` + `_resolve_ref_to_uuid` before `fetch_issue`,
  avoiding a redundant key→UUID round-trip when the ticket is already tracked.
- `_sync_under_lock` now validates URL workspace slug against the manifest
  before any network call (fail-fast), matching the former `_add_under_lock`
  behavior.
- `_sync_under_lock` now detects quarantine-recovery when adding a root and
  logs `"sync: recovered quarantined root ... → active"` or
  `"sync: added root ..."`.
- `_add()` and `_add_under_lock()` deleted entirely (~170 lines removed).
- New `_sync_write_pass` shared helper extracted from the duplicated
  post-traversal pipeline (fetch linked tickets, write, prune, finalize)
  previously inlined in both `_sync_under_lock` and
  `_standalone_sync_under_lock`.
- Module docstring updated to remove `_add` references.

**R2 — Full-rebuild standalone sync**
([src/context_sync/_sync.py](../../src/context_sync/_sync.py)):

- `_standalone_sync_under_lock` rewritten from a one-line delegation to
  `_refresh_under_lock` into a full-rebuild pipeline: pre-fetches all active
  root bundles, builds the reachable graph, then delegates to
  `_sync_write_pass` which fetches all remaining reachable tickets and writes
  unconditionally (skipping only byte-identical rewrites per ADR §8).
- Standalone `sync` now genuinely differs from `refresh`: `sync` re-fetches
  and re-writes all reachable tickets; `refresh` only fetches stale ones.

## Completion Summary
