**Logging audit findings:**

ADR §6.1 requires specific INFO and DEBUG level logging. Audit of existing code
identified these gaps:

1. `sync` completed log was missing `unchanged` count.
2. `refresh` started log was missing `max_tickets_per_root` and used hardcoded
   `"refresh:"` instead of the `snapshot_mode` parameter (so `add`/`remove-root`
   runs incorrectly showed as "refresh:").
3. `refresh` completed log was missing `roots_at_cap` count and also used
   hardcoded `"refresh:"`.
4. `diff` had no start or end INFO logs at all.
5. No DEBUG log for clean lock acquisition.
6. No per-ticket DEBUG logs for fresh/stale decisions during refresh.

**Key decisions:**

1. Extracted the diff pipeline body (~170 code lines) from
   [src/context_sync/_sync.py](../../src/context_sync/_sync.py) into a new
   `run_diff()` async function in
   [src/context_sync/_diff.py](../../src/context_sync/_diff.py) to keep
   `_sync.py` under the 1,000 code-line limit after adding logging.
2. Used `snapshot_mode` parameter in refresh started/completed logs so that
   `add` and `remove-root` operations (which delegate to `_refresh_under_lock`)
   correctly identify themselves in log output.
3. The sample configuration artifact already existed at
   [scripts/.linear_env.sh.sample](../../scripts/.linear_env.sh.sample) and
   satisfies the coding-guidelines "repo-local alternative" requirement. No new
   artifact was needed.
