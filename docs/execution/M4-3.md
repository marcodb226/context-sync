   should be unified to `ticket` so the argparse dest is `args.ticket` across
   all three commands.

2. **CLI banned-term `syncer` variable** — Every command handler in
   [src/context_sync/_cli.py](../../src/context_sync/_cli.py) uses `syncer` as
   the local variable for the `ContextSync` instance. Per
   [docs/policies/terminology.md](../policies/terminology.md), rename to `ctx`.

3. **Testing module `make_syncer`** —
   [src/context_sync/_testing.py](../../src/context_sync/_testing.py)
   exports `make_syncer`. Rename to `make_context_sync`. Update the module
   docstring, section comment, and `FakeLinearGateway` docstring that reference
   the old name.

4. **Test file updates** — All test files that import `make_syncer` and use
   `syncer` as a local variable need updating:
   - [tests/test_cli.py](../../tests/test_cli.py) — arg names and imports
   - [tests/test_e2e.py](../../tests/test_e2e.py) — `_make_args` defaults and imports
   - [tests/test_sync.py](../../tests/test_sync.py) — imports and variable names
   - [tests/test_refresh.py](../../tests/test_refresh.py) — imports and variable names
   - [tests/test_diff.py](../../tests/test_diff.py) — imports and variable names
   - [tests/test_add_remove_root.py](../../tests/test_add_remove_root.py) — imports and variable names

5. **README** — Already uses `TICKET` as the CLI placeholder and
   `ctx.sync(key=...)` in the library example. No changes needed.
