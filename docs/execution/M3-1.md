## Work Log

### Implementation plan

The `refresh()` method implements the incremental whole-snapshot update flow
defined in [docs/adr.md §5.2](../adr.md#52-refresh-incremental-whole-snapshot-update),
[docs/adr.md §6.1](../adr.md#61-snapshot-consistency-contract), and
[docs/design/0-top-level-design.md §6.2](../design/0-top-level-design.md#62-refresh-flow).

It reuses the lock, manifest, traversal, fetch, and write primitives
established by [M2-3](M2-3.md) and adds:

1. **Root visibility check** — batch-query `get_refresh_issue_metadata` for all
   manifest roots to determine visibility.
2. **Quarantine/recovery** — apply `missing_root_policy` to unavailable roots;
   recover previously quarantined roots that are now visible.
3. **Composite cursor freshness** — batch-query remote metadata for all tracked
   reachable tickets, compute remote cursors, compare against local cursors.
4. **Selective re-fetch** — only `fetch_issue` for stale or newly discovered
   tickets.
5. **Pruning** — same derived-ticket pruning as `sync`.

Key design constraints from the implementation plan:
- Implement the refresh contract from [M1-D3](M1-D3.md) exactly.
- Use the adapter boundary from [M1-D2](M1-D2.md).
- `comments_signature` input settled by [M3-O1](M3-O1.md).
- Default `missing_root_policy` is `"quarantine"`; `"remove"` is opt-in.
- Attachment-only drift is outside v1 refresh correctness.

### Implementation

Changed files:

- [src/context_sync/_sync.py](../../src/context_sync/_sync.py) — `refresh()` method and supporting helpers:
  - `_read_existing_ticket_state()` extended to return 3-tuple
    `(refresh_cursor, root_state, format_version)`; all callers updated.
  - `_QUARANTINE_WARNING` constant for quarantine preamble text.
  - `_rewrite_quarantined_ticket()` — surgically updates an existing ticket
    file with quarantine frontmatter and warning preamble without requiring a
    `TicketBundle` (unavailable roots cannot be re-fetched).
  - `refresh()` — public entry point: acquires writer lock, delegates to
    `_refresh_under_lock()`, releases lock in `finally`.
  - `_refresh_under_lock()` — core refresh logic:
    1. Loads existing manifest (raises `ManifestError` if absent).
    2. Batch-checks root visibility via `get_refresh_issue_metadata`.
    3. Applies `missing_root_policy` (`"quarantine"` or `"remove"`) to
       unavailable roots; recovers previously quarantined roots that are
       now visible.
    4. Pre-fetches all active root bundles concurrently.
    5. Builds reachable graph from active roots via `build_reachable_graph`.
    6. Batch-checks freshness: queries `get_refresh_issue_metadata`,
       `get_refresh_comment_metadata`, `get_refresh_relation_metadata` for
       all tracked reachable tickets; computes remote composite cursors and
       compares against local cursors (including `format_version` check).
    7. Fetches only stale and newly discovered tickets.
    8. Writes stale/new tickets; handles root_state consistency for fresh
       tickets whose root state changed (e.g. recovered from quarantine).
    9. Prunes derived tickets no longer reachable from active roots.
    10. Finalizes manifest snapshot with `mode="refresh"`.

- [tests/test_refresh.py](../../tests/test_refresh.py) — 16 integration tests
  across 10 test classes:
  - `TestRefreshUnchangedNoop` (2) — unchanged upstream produces no rewrites;
    manifest `mode` is `"refresh"`.
  - `TestRefreshStaleFresh` (3) — stale ticket updated; comment change triggers
    staleness; relation change triggers staleness.
  - `TestRefreshQuarantine` (2) — unavailable root quarantined with markers;

  - `TestRefreshReactivation` (1) — quarantined root recovered when visible;

  - `TestRefreshSelectiveRewrite` (1) — mixed stale/fresh; only stale rewritten.
  - `TestRefreshNewlyDiscovered` (1) — new reachable ticket discovered via

  - `TestRefreshPruning` (1) — unreachable derived ticket pruned.
  - `TestRefreshRequiresManifest` (1) — raises `ManifestError` without manifest.
  - `TestRefreshMultiRoot` (3) — multi-root refresh; one quarantined / other

### Decisions

- **Quarantine file rewriting without `TicketBundle`**: Quarantined roots cannot
  be re-fetched, so `_rewrite_quarantined_ticket()` reads the existing file,
  updates frontmatter, and injects the warning preamble surgically rather than
  going through `write_ticket()`.
- **Format version as staleness signal**: `_read_existing_ticket_state()` now
  returns `format_version` so refresh can treat tickets written with an older
  format version as unconditionally stale per the design spec.
- **Root_state consistency for fresh tickets**: Even when a ticket's content is
  fresh, if its `root_state` metadata changed (e.g. recovered from quarantine),
  the file is rewritten to keep on-disk state consistent.
